(UF3 Accesibilidad y usabilidad)

**Accesibilidad web**: Acceso universal a la web, que todo el mundo pueda tener acceso a la web independientemente del contexto que se utilice: idioma, cultura, lugar geográfico, etc.

El concepto de accesibilidad dio lugar a la **Iniciativa de Accesibilidad Web (WAI, Web Accesibility Iniciative)**: Su objetivo principal era poder facilitar el acceso a aquellos usuarios que presentaran algún tipo de discapacidad. 

Mediante la utilización de la web se puede ofrecer a aquellas personas que presenten alguna discapacidad una nueva oportunidad de tener acceso a la información. Las webs accesibles representan uno de los recursos más importantes para empleo, educación, sanidad, comercio. Ayudan a que todos los usuarios que desean interactuar con ellas participen más en la sociedad
## 1. Consorcio World Wide Web (W3C)

El **Consorcio World Wide Web (W3C)** es una asociación internacional compuesta por un conjunto de organizaciones que buscan trabajar de forma conjunta para desarrollar los estándares y normas que puede tener una web.

Su **función principal** es guiar a la web hacia su máximo potencial a través del desarrollo de protocolos y pautas que aseguren el crecimiento futuro de la web.

Actúa de siguientes formas:
-  **Mediante el trabajo, potencia la web para todos los usuarios**. Su objetivo principal es conseguir que todos los usuarios accedan a todas las posibilidades planteadas.
- **Facilita el acceso a la web desde cualquier tipo de dispositivo** independientemente de la forma de itneractuar usada (formulario, gestos, voz, etc)
- **Ayuda y apuesta por una buena calidad en los contenidos** que presenta la web
- **Colabora con la evolución que va adquiriendo la web** aumentando sus posibilidades en tres dimensiones diferentes: **Colaboración (web 2.0)**, **semántica (web 3.0)**, **universal (web 4.0)**
- **Añade nuevos mecanismos que garantizan seguridad, protección y cifrado** de la información disponible.
## 2. Principios generales de diseño accesible

Para conseguir que los sitios webs sean accesibles hay una serie de principios de diseño que el consorcio W3C ha identificado y fijado.
En 1999, las guías de accesibilidad del contenido web **WCAG 1.0** (Web Content Accessibility Guidelines) indicaron catorce principios:
1.  **Suministrar una serie de alternativas para los contenidos**, tanto visuales como auditivos
2. **No centrarse tanto en el color**, hay dispositivos que no los pueden visualizar
3. **Hacer uso de marcadores y hojas de estilo de forma adecuada**
4. **Reconocer el lenguaje natural que se utilice**. Si no se reconoce pueden producirse errores. Por ejemplo, las abreviaturas no se podrán traducir por los dispositivos Braille o lectores de pantalla
5. **Al crear tablas**, asegurarse de que son fáciles de transformar a través de las diferentes aplicaciones y de los navegadores accesibles
6. **Comprobar que las páginas que hacen uso de las nuevas tecnologías** se pueden transformar correctamente
7. **Comprobar que todos los objetos que se encuentran en movimiento están controlados**
8. **Garantizar la accesibilidad directa** de las diferentes interfaces que se utilicen
9. **Al realizar el diseño, tener en cuenta todos los dispositivos (ratón, teclado, voz, etc.)**
10. **Tener disponibilidad de las soluciones provisionales**
11. **Hacer uso de tecnologías y pautas W3C**
12. Ayudar a los usuarios **a entender las páginas mediante orientación e información de contexto**
13. Ayudar a los usuarios a **encontrar la información que buscan a través de la utilización de mecanismos claros y consistentes**
14. Mediante el uso de **documentos claros y sencillos** garantizar la comprensión de los usuarios

En 2009, **WCAG 2.0** reduce los catorce principios a cuatro:
- **ROBUSTO**: Sitios web fuertes para trabajar con diferentes agentes de usuarios y tecnologías necesarias para personas que presenten alguna discapacidad. 
- **FÁCIL DE COMPRENDER o COMPRENSIBLE**: Sitios web sencillos de comprender para los usuarios
- **FÁCIL DE OPERAR u OPERABLE**: Sitios web sencillos para operar de forma sencilla
- **FÁCIL DE PERCIBIR O PERCEPTIBLE**: Sitios web fuertes sencillos de concebir para que los usuarios puedan navegar sin problema.
## 3. Pautas de accesibilidad al contenido en la web (WCAG)

Siguen los cuatro principios mencionados (Robusto, Comprensible, Operable, Perceptible) y engloban todas las propiedades que garantizan un contenido web accesible. 

**Siguiendo estas pautas se garantiza que el contenido sea accesible para más usuarios** con discapacidad como ceguera, visión insuficiente, pérdida de audición y sordera, deficiencias del lenguaje, etc.

E**s conveniente que el diseñador conozca estos principios y los ponga en práctica** para que los sitios web lleguen a muchos más usuarios.

| Principios de diseño accesible | Pautas de accesibilidad 2.0                                                                                                                                                                                                                                                                                                                                                                                                                |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Robusto**                    | - Aumentar la **compatibilidad de los agentes de usuarios actuales con los que se incorporen después**                                                                                                                                                                                                                                                                                                                                     |
| **Comprensibilidad**           | - Necesario contenido textual **legible y comprensible**<br>- **Páginas webs intuitivas** según su apariencia<br>- **Ofrecer ayuda a los usuarios** para evitar y corregir errores                                                                                                                                                                                                                                                         |
| **Operatividad**               | - Permitir **acceder a las funcionalidades mediante el teclado**<br>- **Ofrecer a los usuarios con discapacidad tiempo suficiente** para poder leer el contenido y utilizarlo<br>- **No** realizar **diseño** **que pueda provocar algún ataque**<br>- **Ofrecer una serie de medios que sirvan de ayuda a los usuarios que tengan alguna discapacidad** a la hora de navegar, buscar contenido y visualizar dónde está ubicado            |
| **Perceptibilidad**            | - **Presentar diferentes opciones de contenido no textual** para adaptarse a las necesidades de las personas<br>- **Disponer de alternativas para contenidos multimedia** que dependen del tiempo<br>- **Permitir la creación de contenido para presentarlo de diferentes formas** sin que se pierda la estructura y la información<br>- **Facilitar a los usuarios que interactúen en el sitio web**: ver y oir el contenido que contiene |

## 4. Técnicas para satisfacer los requisitos definidos en las WCAG

Para establecer las pautas de accesibilidad al contenido web 2.0, se ha diseñado un documento que cubra las necesidades de todos aquellos que necesiten un estándar técnico estable que garantice la accesibilidad a un sitio web.

Además, existen documentos basados en las pautas 2.0 que cumplen una serie de requisitos adicionales que asisten a:
- **El cumplimiento de las pautas 2.0**: Listado de todas las pautas, criterios de éxito y diferentes técnicas que se pueden utilizar en el desarrollo del contenido web
- **La puesta en práctica de las pautas 2.0**: Documento que contiene la comprensión de las diferentes pautas junto con sus correspondientes criterios de éxito 
- **Las técnicas para satisfacer las pautas 2.0:** Conjunto de las diferentes técnicas conocidas y de los errores junto con una descripción de estos.
- **La comprensión de las propias pautas 2.0:** Diagrama en el que identificamos la relación existente entre los diferentes criterios técnicos.

Los documentos definidos van a ofrecer una serie de criterios de éxito que pueden usarse especificar un diseño, una compra, el cumplimiento de una regulación...
Para cumplir estos requisitos están los niveles A (más bajo), AA, AAA (más alto)

Cada letra indica qué tan accesible es algo:

- **Nivel A**: el mínimo. Si no cumples esto, tu contenido es bastante inaccesible.
- **Nivel AA**: es el nivel estándar que debería cumplirse en casi todos los sitios web (por ley en muchos países).
- **Nivel AAA**: es el más alto. Se aplica solo en contextos muy exigentes (como webs muy inclusivas o específicas para discapacidad).

Veamos las **pautas asociadas asociadas a cada criterio y al nivel de adecuación**

| Pauta                                           | Criterios para tener en cuenta (nivel de adecuación)                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| ----------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Alternativas textuales**                      | Hay diferentes formas de presentar la información y diferentes mecanismos para interactuar.<br>Para que sea accesible, todo contenido no textual debe contar con una alternativa textual que logre un propósito equivalente, excepto determinados supuestos (véase WCAG 2.0 (A))                                                                                                                                                                                                                                        |

## 5. Métodos para realizar revisiones preliminares y evaluaciones de adecuación o conformidad de documentos web

El diseñador, siguiendo las pautas recomendadas, diferencia entre dos actividades diferentes:
- **Dirigir el proceso de desarrollo de los diferentes sitios web**: Asegurarse de que un determinado sitio web cumple los criterios de accesibilidad recomendados. 
- **Comprobar los diseños web**: En función de los criterios de accesibilidad utilizados.

Se muestran ahora después una serie de herramientas que permiten analizar de forma sencilla un sitio web y comprobar que cumple los requisitos para que sea accesible.
1. Seleccionamos una pequeña muestra de diferentes páginas
2. Mediante navegadores gráficos, evaluamos las páginas con sus correspondientes configuraciones
3. Utilizamos navegadores de voz o de texto para que todas las personas puedan tener acceso al sitio web
4. Utilizamos herramientas de evaluación de accesibilidad
5. Realizamos un resumen de los resultados obtenidos

## 6. Herramientas de análisis de accesibilidad web

Hay numerosas herramientas que permiten saber si existen problemas de accesibilidad aunque no hay ninguna que lo indique al 100% --> **Esta tarea no se puede realizar de forma automática**

Hay herramientas que pretenden automatizar el proceso de evaluación de la accesibilidad pero siempre suelen dar algún tipo de error (falsos positivos) que debe ser resuelto de forma manual por parte del usuario. 

**Recomendable** añadir análisis con evaluaciones manuales a estas herramientas de análisis y evaluación automáticas. Así podrán tratarse los falsos positivos.

## 7. Revisión de la accesibilidad web desde diferentes navegadores

La mayoría de navegadores permiten facilitar el proceso de evaluación de la accesibilidad de una web de forma manual mediante una serie de extensiones (permiten modificar la resolución de forma rápida, comprobar los colores de un sitio web, etc...). Alguna de las extensiones pueden ser:
- **Web Developer**: Barra de herramientas usada en Mozilla Firefox
- **Firefox Accesibility Extension**: Extensión para la barra de herramientas usada en Mozilla Firefox con las diferentes opciones que facilitan la navegación por contenidos de usuarios con alguna discapacidad.
- **Web Accesibility Toolbar**: Plugin utilizado en Internet Explorer que permite una evaluación manual sencilla de la accesibilidad de un sitio web.
- **Firebug**: Extensión que permite que los desarrolladores puedan realizar modificaciones sobre el código fuente HTML, CSS, JavaScript 
- **Fangs**: Extensión que permite mostrar el contenido de una determinada página como si fuese un lector de pantalla
- **HTML Validator Tidy**: Extensión para Mozilla que permite agregar un validador de HTML en el interior de Firefox.
